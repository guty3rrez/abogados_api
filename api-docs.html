API de Reservas - Documentación

Esta API permite consultar horarios disponibles y realizar reservas para citas.

La API responde en formato JSON.

La URL base dependerá de si el servidor se ejecuta en modo HTTP o HTTPS y del puerto configurado:

  • Modo HTTP: http://[TU_DOMINIO_O_IP]:[PUERTO_HTTP] (Ej: http://localhost:3000)
  • Modo HTTPS: https://[TU_DOMINIO_O_IP]:[PUERTO_HTTPS] (Ej: https://localhost:443 o https://localhost)

Nota Importante: Para ejecutar en modo HTTPS, se requieren los archivos key.pem y cert.pem en el mismo directorio que el script del servidor.

Autenticación

Actualmente, la API no requiere autenticación.

Códigos de Estado Comunes

  • 200 OK: Solicitud exitosa (generalmente para GET).
  • 201 Created: Recurso creado exitosamente (generalmente para POST).
  • 400 Bad Request: La solicitud tiene un formato incorrecto, faltan parámetros o los datos no son válidos. El cuerpo de la respuesta contendrá detalles del error.
  • 404 Not Found: El endpoint solicitado no existe.
  • 409 Conflict: Se intentó crear un recurso que ya existe (ej: reservar un horario ya ocupado).
  • 500 Internal Server Error: Ocurrió un error inesperado en el servidor.

Endpoints

GET /slots

Obtiene los horarios de reserva disponibles para una fecha específica.

Parámetros de Consulta (Query Parameters)

Parámetro Tipo Descripción Requerido
date String La fecha para la cual consultar la disponibilidad. Formato: YYYY-MM-DD. Solo se permiten días laborables (Lunes a Viernes). Sí

Horarios de Atención (Business Hours)

Los horarios disponibles inicialmente son:

["09:00", "10:00", "11:00", "12:00", "14:00", "15:00", "16:00", "17:00"]

La API filtra estos horarios eliminando los ya reservados y, si la consulta es para el día actual, los horarios que ya han pasado.

Respuesta Exitosa 200 OK

Devuelve un objeto JSON con una lista de los horarios disponibles.

{
  "available_slots": [
    "14:00",
    "15:00",
    "16:00",
    "17:00"
  ]
}

Si no hay horarios disponibles para la fecha seleccionada (o la fecha ya pasó), la lista estará vacía:

{
  "available_slots": []
}

Respuestas de Error

  • 400 Bad Request - Parámetro 'date' faltante: 
    { "error": "Missing 'date' query parameter." }
  • 400 Bad Request - Formato de fecha inválido: 
    { "error": "Invalid date format. Please use YYYY-MM-DD." }
  • 400 Bad Request - Fecha no es día laborable: 
    { "error": "Bookings are only available on weekdays (Monday to Friday)." }
  • 500 Internal Server Error - Error interno del servidor: 
    { "error": "Internal server error retrieving slots." }

POST /book

Crea una nueva reserva para una fecha y hora específicas.

Cuerpo de la Solicitud (Request Body)

Debe ser un objeto JSON con la siguiente estructura:

Campo Tipo Descripción Requerido Validación
selected_date String Fecha de la reserva. Sí Formato YYYY-MM-DD, debe ser un día laborable (Lun-Vie), no puede ser una fecha pasada.
selected_time String Hora de la reserva. Sí Debe ser uno de los horarios disponibles devueltos por GET /slots para esa fecha. No puede ser una hora pasada en el día actual.
nombre String Nombre completo de quien reserva. Sí No puede estar vacío.
email String Dirección de correo electrónico. Sí Debe ser un formato de email válido (ej: usuario@dominio.com).
telefono String Número de teléfono de contacto. Sí No puede estar vacío.
motivo String Motivo de la reserva (opcional). No -
Ejemplo de Cuerpo de Solicitud:
{
  "selected_date": "2024-08-15",
  "selected_time": "11:00",
  "nombre": "Juan Pérez",
  "email": "juan.perez@example.com",
  "telefono": "123456789",
  "motivo": "Consulta general"
}

Respuesta Exitosa 201 Created

Devuelve un objeto JSON confirmando la reserva e incluyendo algunos detalles.

{
  "success": true,
  "message": "Your booking has been successfully confirmed!",
  "booking": {
    "id": 15, // ID generado por la base de datos
    "date": "2024-08-15",
    "time": "11:00",
    "name": "Juan Pérez"
  }
}

Respuestas de Error

  • 400 Bad Request - Fallo de validación:

    Devuelve un objeto JSON detallando los campos con errores.

    {
  "message": "Validation failed.",
  "errors": {
    "selected_date": "Valid booking date (YYYY-MM-DD) is required.",
    "email": "Valid email address is required.",
    "selected_time": "Cannot book a time slot that has already passed today."
    // ... otros posibles errores
  }
}
  • 409 Conflict - Horario ya reservado: 
    {
  "message": "Sorry, this time slot was just booked or is already taken. Please choose another one."
}
  • 500 Internal Server Error - Error interno del servidor: 
    { "message": "Internal server error processing booking." }

Base de Datos

La API utiliza una base de datos PostgreSQL. La tabla principal es reservations con la siguiente estructura (aproximada):

CREATE TABLE reservations (
    id SERIAL PRIMARY KEY,
    fecha DATE NOT NULL,
    hora VARCHAR(5) NOT NULL,
    nombre TEXT NOT NULL,
    email TEXT NOT NULL,
    telefono TEXT NOT NULL,
    motivo TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (fecha, hora) -- Asegura que cada horario sea único por fecha
);

La conexión se configura mediante la variable de entorno DATABASE_URL.
